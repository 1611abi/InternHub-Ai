# Project Architecture & Documentation: CareerVault / JobQuest

This document provides a comprehensive overview of the project structure, the purpose of each component, and the technical implementation details for your review and presentation.

---

## 📂 Folder & File Details

### 1. Root Directory
- `backend/`: The server-side application (Node.js).
- `frontend/`: The client-side user interface (React).
- `ml-service/`: The Machine Learning processing engine (Python).
- `uploads/`: Temporary storage for uploaded resumes and resources.
- [package.json](file:///d:/Desktop/my-frist-app/package.json): Main project configuration for dependencies.

---

### 2. Backend (`/backend`)
The backend acts as the "brain," managing data flow, security, and communication with the ML models.

- **`server.js`**: The main entry point. Sets up the Express server, connects to MongoDB, and initializes all routes.
- **`models/`**: Defines the "Shape" of your data in the database (Mongoose Schemas).
  - `User.js`: Stores user profiles (Job Seekers, Recruiters, Admins).
  - `Job.js` & `Internship.js`: Stores job postings.
  - `Resume.js` & `ATSReport.js`: Stores uploaded resumes and the scores generated by ML.
  - `careervault/`: Models for community-shared resources (Resource, Folder, Category).
- **`controllers/`**: Contains the actual logic for processing requests.
  - `authController.js`: Handles Registration, Login, and Password hashing.
  - `atsController.js`: Manages the flow of sending resumes to the ML-service and saving the results.
  - `careervaultController.js`: Handles file uploads, moderation (Approve/Reject), and resource management.
  - `recruiterController.js`: Manages job postings and applicant tracking for recruiters.
- **`routes/`**: Defines the API endpoints (e.g., `GET /api/jobs`, `POST /api/ats/analyze`).
- **`middleware/`**: Functions that run "in the middle" (e.g., `auth.js` to check if a user is logged in before allowing access).

---

### 3. ML-Service (`/ml-service`)
This is where the heavy lifting of data analysis happens using Artificial Intelligence and NLP (Natural Language Processing).

- **`app.py`**: The Flask server entry point that provides APIs for the backend to call.
- **`ats_model.py`**: Uses **NLP** (spaCy) and **Semantic Similarity** (Sentence-Transformers) to extract keywords and calculate a match score. It also uses **TF-IDF Vectorization** to check keyword density against industry standards.
- **`job_recommender.py`**: A recommendation engine that uses **Cosine Similarity** to suggest the best-fit jobs based on a user's profile.
- **`skill_gap_model.py`**: Analyzes missing skills by comparing the user's extracted skills with job requirements using **Keyword Matching**.
- **`requirements.txt`**: Lists the Python libraries needed (`spacy`, `scikit-learn`, `sentence-transformers`, `flask`, `pandas`).

---

### 4. Frontend (`/frontend`)
The visual part of the website built using **React** and **Tailwind CSS**.

- **`src/App.jsx`**: The main controller for the UI, handling navigation and routing.
- **`src/pages/`**: Complete screens of the application.
  - `careervault/`: Resource library pages.
  - `admin/`: Dashboard for site administrators to moderate content.
  - `seeker/` & `recruiter/`: Role-specific dashboards.
- **`src/components/`**: Small, reusable parts of the UI (Buttons, Navbar, Modals, Cards).
- **`src/services/`**: The "bridge" to the backend. Files like `careervaultApi.js` use `axios` to fetch data from your Node.js server.
- **`src/context/`**: Manages "Global State" like whether a user is logged in or which theme (Dark/Light) is active.

---

## 🔗 How Parts are Connected (The Data Flow)

### 🔄 The "ATS Analysis" Flow:
1. **User Action**: A user uploads a Resume PDF on the **Frontend**.
2. **Backend Entry**: The **Frontend** sends the file to the **Backend** (`atsController.js`).
3. **Storage**: The **Backend** saves the file and gets its path.
4. **ML Request**: The **Backend** makes an HTTP POST request to the **ML-Service** (`app.py` at `localhost:5000`).
5. **AI Processing**: The **Python ML model** reads the text, extracts skills, and calculates a percentage. It sends this JSON data back to the **Backend**.
6. **Persistence**: The **Backend** saves this "ATS Report" in MongoDB linked to the user.
7. **Display**: The **Backend** sends the result to the **Frontend**, which triggers a "Success" animation and shows the score.

## 🎤 How to Explain Your Project

When presenting, use the **"Problem -> Solution -> Impact"** framework:

### 1. The Problem
"Job seekers often don't know why their resumes are getting rejected by automated systems, and they struggle to find quality resources or internships tailored to their specific skills."

### 2. The Solution (Your Project)
"I built an integrated Career Platform that uses a **triple-layered architecture**:
- **A Modern UI** (React/Tailwind) for a seamless user experience.
- **A Secure Backend** (Node.js/MongoDB) to manage data and role-based access.
- **An Intelligent ML Engine** (Python) that performs ATS scoring, skill-gap analysis, and job recommendations."

### 3. Key Feature Highlights
- **ATS Optimizer**: Explain that it uses NLP to give users real feedback on their resumes.
- **CareerVault**: Mention the community-driven resource library with an Admin moderation system to ensure quality.
- **Role-Based Dashboards**: Highlight that Recruiters and Job Seekers have completely different tools (e.g., Recruiters track applicants, Seekers find jobs).

### 4. Technical Stack
- **Frontend**: React, Tailwind CSS, Framer Motion (animations), Vite, Axios.
- **Backend**: Node.js, Express, MongoDB (NoSQL), JWT (Authentication), Multer (File Uploads).
- **ML**: Python, Flask, spaCy (NLP), Sentence-Transformers (BERT/Similarity), Scikit-learn (TF-IDF).
